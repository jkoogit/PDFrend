# [프로세스 정의서] PDFrend: 데이터 흐름 및 시스템 로직 (v2.0)

본 문서는 [시스템 정책서](./01_Policy_Management.md)에 정의된 정책들을 기반으로 합니다.

## 1. 문서 업로드 및 리소스 분리 프로세스
*   **용량 검증**: [Policy-STG-002] 업로드 전 사용자의 잔여 용량을 확인하여 초과 시 업로드 차단 및 안내 메시지 노출.
1.  **파일 수신**: PDF 업로드 시 파일명 기반 도서 식별.
2.  **비동기 리소스 추출**:
    *   **이미지**: 페이지별 JPG 추출 (Lazy Loading 지원을 위해 저해상도/고해상도 구분 생성 검토).
    *   **OCR**: 텍스트 좌표 및 내용 추출 -> JSONB 저장.
    *   **주석/목차**: 기존 PDF 내 Annotation 및 Outline 정보 추출 -> UUID 부여 후 JSONB 저장.
3.  **메타데이터 기록**: 문서 내부에 `Document_ID`, `Initial_Version`, `Owner` 정보 기록.

---

## 2. 실시간 동기화 및 버전 생성 프로세스
*   **버전 관리**: [Policy-VER-001], [Policy-VER-002], [Policy-VER-003] 정책 적용.
1.  **주석 편집**: 작업자가 주석 수정 시 WebSocket을 통해 서버로 `Delta` 데이터 전송.
2.  **서버 검증**: 
    *   작성자 권한 체크 (본인 주석 여부 또는 소유자 여부).
    *   충돌 체크 (동일 주석에 대한 동시 수정 여부).
3.  **버전 생성 (Checkpoint)**:
    *   소유자가 버전 생성 클릭 -> 현재 주석 상태를 `Snapshot` 테이블로 복사.
    *   타 작업자들에게 `Version_Updated` 이벤트 전송 -> 클라이언트는 내부적으로 현재 작업 버전을 업데이트하고 계속 진행.

---

## 3. 오프라인 동기화 및 머지 프로세스
*   **충돌 해결**: [Policy-OFF-001], [Policy-OFF-002] 정책 적용.
1.  **오프라인 작업**: 네트워크 단절 시 모든 주석 변경 사항을 IndexedDB에 `Pending_Action` 리스트로 저장.
2.  **온라인 전환**: 서버와 메타데이터(Timestamp, Version) 비교.
3.  **변경점 선택 머지 (Selective Merge)**:
    *   서버 버전과 로컬 버전의 차이점을 분석하여 리스트업.
    *   사용자가 각 변경 항목(추가, 수정, 삭제)을 확인하고 최종 반영 버튼 클릭.
    *   서버는 선택된 항목들을 현재 최종 버전에 병합하고 새로운 `Timestamp` 부여.

---

## 4. 공유 및 권한 실시간 제어 프로세스
1.  **권한 변경**: 소유자가 특정 사용자의 권한을 '쓰기'에서 '읽기'로 변경.
2.  **실시간 전파**: 서버가 해당 사용자의 세션에 `Permission_Changed` 신호 전송.
3.  **클라이언트 반응**: 
    *   뷰어의 주석 도구바 즉시 비활성화.
    *   "편집 권한이 회수되었습니다. 읽기 모드로 전환합니다." 알림 표시.
    *   진행 중이던 미저장 작업은 `Draft`로 임시 보관 처리.

---

## 5. 다운로드 및 병합 프로세스
1.  **병합 엔진 가동**: 이미지 + OCR(투명 레이어) + 주석(표준 Annotation) 병합.
2.  **Xodo 호환성 확보**: 
    *   주석 타입을 Xodo 표준(FreeText, Highlight, Stamp 등)으로 매핑.
    *   PDF 메타데이터(`Info Dictionary`)에 버전 및 수정 이력 필드 삽입.
3.  **파일명 생성**: `[도서명]_[버전]_[YYYYMMDD]_[HHMMSS].pdf` 포맷으로 최종 파일 생성 및 전송.
