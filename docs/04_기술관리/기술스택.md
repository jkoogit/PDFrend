# 기술 스택 및 구현 가이드

## 📌 개요

PDFrend 백엔드는 **Node.js + TypeScript + tRPC** 스택을 기반으로 구축되었습니다. 이 문서는 사용 중인 모든 기술, 버전, 그리고 구현 가이드를 정리합니다.

## 🔧 개발 환경 설정

### 필수 설치

```bash
# Node.js 설치 (22.13.0 이상)
node --version

# pnpm 설치
npm install -g pnpm

# 의존성 설치
pnpm install

# 환경 변수 설정
cp .env.example .env.local
```

### 개발 서버 실행

```bash
# 백엔드 + 프론트엔드 동시 실행
pnpm dev

# 백엔드만 실행
pnpm dev:server

# 프론트엔드만 실행
pnpm dev:client
```

### 빌드

```bash
# 전체 빌드
pnpm build

# 백엔드 빌드
pnpm build:server

# 프론트엔드 빌드
pnpm build:client
```

### 파일 구조

```
src/
├── server/
│   ├── domains/           # 도메인 레이어
│   ├── applications/      # 애플리케이션 레이어
│   ├── infrastructure/    # 인프라 레이어
│   ├── routers.ts         # tRPC 라우터
│   ├── db.ts              # DB 쿼리 헬퍼
│   └── _core/             # 프레임워크 코어
├── client/
│   ├── src/
│   │   ├── pages/         # 페이지 컴포넌트
│   │   ├── components/    # 재사용 컴포넌트
│   │   ├── lib/           # 유틸리티
│   │   ├── App.tsx        # 라우팅
│   │   └── main.tsx       # 진입점
│   └── index.html
├── drizzle/               # DB 스키마
├── shared/                # 공유 타입
└── storage/               # 파일 저장소 헬퍼
```

### 명명 규칙

**파일명:**
- 컴포넌트: `PascalCase` (예: `UserProfile.tsx`)
- 유틸리티: `camelCase` (예: `formatDate.ts`)
- 타입: `PascalCase` (예: `User.ts`)
- 테스트: `*.test.ts` (예: `User.test.ts`)

**변수명:**
- 상수: `UPPER_SNAKE_CASE` (예: `MAX_FILE_SIZE`)
- 변수/함수: `camelCase` (예: `getUserById`)
- 클래스: `PascalCase` (예: `UserService`)
- 인터페이스: `I` 접두사 (예: `IUserRepository`)

### 환경 변수

민감한 정보는 환경 변수로 관리합니다:
- `DATABASE_URL` - 데이터베이스 연결 문자열

- `API_KEY` - 외부 API 키

### 입력 검증

모든 입력은 Zod로 검증합니다:

```typescript
const userSchema = z.object({
  name: z.string().min(1).max(100),
  email: z.string().email(),
  age: z.number().int().min(0).max(150),
});
```
