# 09. 간편인증 단독도입(휴대폰 본인인증 제외) 시 동일인 식별 시나리오 검토

## 1. 질문 요약
- 휴대폰 본인인증 없이 카카오/네이버/구글 간편인증만으로 운영 가능한가?
- 타 기관(예: 카카오 가입자와 네이버 가입자) 간 동일인 식별이 가능한가?
- 불가능/불완전하다면, 이메일 인증 기반으로 운영 대안이 가능한가?

## 2. 핵심 결론
1. **도입 가능**: 휴대폰 본인인증 없이도 간편인증 로그인은 가능.
2. **완전한 동일인 식별은 어려움**: 기관별 식별자 체계가 다르므로 자동 통합은 위험.
3. **실무 대안**: "기관별 인증키 + 필수 이메일 인증 + 로그인 후 계정연결" 방식이 가장 현실적.

---

## 3. 제공사 API 관점 식별자 특성

## 3.1 Google
- OIDC에서 `sub`를 계정의 안정 식별자로 사용 권장.
- 이메일은 변경 가능성이 있으므로 식별자 기본키로 단독 사용 비권장.
- 시사점: `iss + sub`를 내부 identity 키로 저장.

## 3.2 Kakao
- 카카오 로그인 문서/FAQ 기준 서비스 사용자 ID(user id) 제공.
- 앱/서비스 설정에 따라 사용자 ID 고정 옵션 고려 필요.
- 시사점: 내부에 `provider=kakao + provider_key_val=user_id`로 저장.

## 3.3 Naver
- 네이버 로그인에서도 공급자 고유 식별값(id) 기반 연동이 일반적.
- 시사점: 내부에 `provider=naver + provider_key_val=id` 저장.

---

## 4. 동일인 식별 가능/불가 판단

### 4.1 가능한 것
- "같은 기관" 내 동일인 식별: 가능
  - 예: 카카오 A 사용자 재로그인 -> 동일 `provider_key_val`로 식별

### 4.2 어려운 것
- "기관 간" 자동 동일인 식별: 원칙적으로 어려움
  - 카카오 식별자와 네이버 식별자는 직접 대응되지 않음
  - 이메일이 같아도 자동 병합은 오탐 리스크 존재

---

## 5. 제안하신 시나리오 검토

### 시나리오
1) 카카오 간편인증 회원가입(카카오 인증키 등록)  
2) 이메일은 서비스 소식/공동작업 알림용으로 등록(인증 선택)  
3) 카카오 로그인 가능  
4) 로그인 후 사용자정보에서 네이버 인증 추가 등록  
5) 네이버 로그인 가능  
6) 인증수단 등록/삭제 관리  
7) 이메일은 알림 채널로 수정/인증 관리

### 검토 결과
- **매우 현실적인 운영안**이며, 휴대폰 본인인증 없이도 관리 가능.
- 핵심은 "자동 병합"이 아니라 **로그인 세션 기반 명시적 연결(Link)**.

### 필수 제어 규칙
1. 최초 가입 시: `provider_key_val` 저장을 가입 완료 조건으로 사용 (이메일 인증은 로그인 필수 조건 아님)
2. 추가 인증 연결 시: 이미 로그인된 회원 세션에서만 연결 허용
3. 연결 충돌 시: 다른 회원에 연결된 인증키면 연결 차단 + 안내
4. 이메일 변경 시: 알림 신뢰성을 위해 재인증 권장(로그인 필수 조건 아님)
5. 인증수단 삭제 시: 최소 1개 로그인 수단 유지 강제

---

## 6. 권장 데이터/정책 포인트
- `auth_identities(provider_cd, provider_key_val)` UNIQUE
- `user_auth_links(user_id, auth_id)` UNIQUE
- `users.email_addr`는 공동작업/서비스 소식 전달용 채널로 사용(로그인 식별과 독립)
- 계정 병합은 자동 금지, 수동 검증 플로우(본인확인 질문/재로그인 확인)로 한정

## 7. 운영 리스크 및 대응
1. 이메일 동일인 오탐 병합 리스크
   - 대응: 이메일 동일만으로 자동 링크 금지
2. 인증수단 탈퇴/차단 리스크
   - 대응: 다중 인증수단 연결 유도(최소 2개 권장)
3. 계정 잠금 리스크
   - 대응: 이메일 인증 수단을 항상 복구 채널로 유지

## 8. 최종 권고
- **휴대폰 본인인증 없이도** 간편인증 운영은 가능하나,
- 동일인 식별은 "기관별 키 + 로그인 후 명시적 연결"로 설계해야 안전하다.
- 이메일은 인증수단이 아니라 협업/공지 커뮤니케이션 채널로 분리 운영한다.
