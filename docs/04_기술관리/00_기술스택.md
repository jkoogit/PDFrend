# 기술 스택 - 현행화 (Node.js 기반)

## 📌 개요

**업데이트 날짜:** 2026-05-12  
**이전 계획:** Kotlin + Spring Boot (코프링)  
**현재 결정:** Node.js + TypeScript + Express + tRPC  
**인프라:** NAS/Ubuntu 로컬 디스크 (비용 최소화)

---

## 🔄 기술 스택 변경 사유

### 이전 계획 (코프링)의 문제점
- 학습 곡선 높음
- 초기 설정 복잡성
- 개발 속도 느림
- 운영 부담 증가

### 현재 결정 (Node.js)의 장점
- **빠른 개발 속도**: JavaScript/TypeScript의 생산성
- **비용 최소화**: NAS/Ubuntu 로컬 디스크 사용으로 인프라 비용 절감
- **타입 안전성**: TypeScript로 런냀임 오류 감소
- **tRPC**: 타입 안전한 RPC 통신
- **학습 용이**: 팀 내 JavaScript 경험 활용

## 🚀 개발 환경 설정

### 필수 설치

```bash
# Node.js 설치 (22.13.0 이상)
node --version

# pnpm 설치
npm install -g pnpm

# 의존성 설치
pnpm install

# 환경 변수 설정
cp .env.example .env.local
```

### 개발 서버 실행

```bash
# 백엔드 + 프론트엔드 동시 실행
pnpm dev

# 백엔드만 실행
pnpm dev:server

# 프론트엔드만 실행
pnpm dev:client
```

```bash
# 전체 빌드
pnpm build

# 백엔드 빌드
pnpm build:server

# 프론트엔드 빌드
pnpm build:client

```

```
├── client/
│   ├── src/
│   │   ├── pages/         # 페이지 컴포넌트
│   │   ├── components/    # 재사용 컴포넌트
│   │   ├── lib/           # 유틸리티
│   │   ├── App.tsx        # 라우팅
│   │   └── main.tsx       # 진입점
│   └── index.html
├── drizzle/               # DB 스키마
├── shared/                # 공유 타입
└── storage/               # 파일 저장소 헬퍼
```

### 명명 규칙

**파일명:**
- 컴포넌트: `PascalCase` (예: `UserProfile.tsx`)
- 유틸리티: `camelCase` (예: `formatDate.ts`)
- 타입: `PascalCase` (예: `User.ts`)
- 테스트: `*.test.ts` (예: `User.test.ts`)

**변수명:**
- 상수: `UPPER_SNAKE_CASE` (예: `MAX_FILE_SIZE`)
- 변수/함수: `camelCase` (예: `getUserById`)
- 클래스: `PascalCase` (예: `UserService`)
- 인터페이스: `I` 접두사 (예: `IUserRepository`)

## 🔒 보안

### 환경 변수

민감한 정보는 환경 변수로 관리합니다:
- `DATABASE_URL` - 데이터베이스 연결 문자열
- `JWT_SECRET` - JWT 서명 키
- `VITE_APP_ID` - Manus OAuth 앱 ID
- `OAUTH_SERVER_URL` - OAuth 서버 URL

### 입력 검증

모든 입력은 Zod로 검증합니다:

```typescript
const userSchema = z.object({
  name: z.string().min(1).max(100),
  email: z.string().email(),
  age: z.number().int().min(0).max(150),
});
```
