# 07. 본인인증 서비스 종류/비용/API 비교 가이드

## 1. 목적
소셜 간편인증 외에 적용 가능한 본인인증 서비스를 정리하고, **API 제공 형태/절차/비용 공개 여부**를 비교한다.

## 2. 빠른 결론
- 국내 본인인증(B2B) 서비스는 대부분 **요금 비공개(영업 문의형)** 이다.
- "무료 상용" 본인인증은 사실상 어렵고, 테스트/개발 단계 무료 크레딧이 있을 수 있다.
- 구현 난이도/확장성 관점에서는 **통합 본인인증 게이트웨이(예: PortOne) + 직접 사업자 계약** 조합이 운영 효율이 높다.

## 3. 서비스 종류 분류
1. **통합 본인인증 게이트웨이형**
   - 여러 인증수단(PASS/네이버/카카오/금융인증서 등)을 단일 API/SDK로 연동
2. **PG/인증사 직접 연동형**
   - KG이니시스, NICE, KCB 등 사업자별 API를 직접 연동
3. **해외 SMS OTP API형**
   - Twilio/AWS SNS 등으로 직접 SMS OTP 구축(국내 법규/발신번호 이슈 별도 검토 필요)

| 구분 | 대표 서비스 | API/문서 | 제공 인증수단(예시) | 비용 공개 | 비고 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 통합형 | PortOne 통합 본인인증 | 문서/FAQ 공개 | 네이버/PASS/페이코/TOSS/카카오/금융인증서 등 | 비공개(문의) | 카카오 인증의 CI 제공 제약 안내됨 |

## 5. 적용 절차(권장)

## 5.1 통합형(PortOne 등) 기준
1. 본인인증 수단 선정(PASS/네이버/카카오 등)
2. 통합사업자 계약 및 운영키 발급
3. 서버에서 인증요청 생성(트랜잭션 ID, callback URL, state)
4. 프론트 인증창/앱 호출
5. 콜백 결과 수신 후 서버 검증(API 재검증)
6. 내부 `user.uuid`와 인증수단 링크 저장

## 5.2 직접형(인증사 직접)
1. 사업자/서비스 심사 및 계약
2. 연동 스펙/테스트 계정 수령
3. 인증요청/결과검증 API 직접 구현
4. 장애/요금/정산/CS 프로세스 개별 운영

## 5.3 SMS OTP 직접 구축형
1. SMS 발송 API 계약(Twilio/AWS 등)
2. OTP 생성/TTL/횟수제한/재시도 정책 구현
3. 번호 재할당/중복가입/알림중복 정책 구현
4. 비용 모니터링(국가별 단가, 실패율) 구축

## 6. 비용 관련 실무 체크리스트
- 건당 과금(국내/해외) 단가
- 월 기본료/연동비/심사비 유무
- 실패 건 과금 기준
- 발신번호 등록/임대 비용
- 규제 대응 비용(A2P 등록 등)
- 정산 주기/세금계산서 발행

## 7. "무료" 가능성 정리
- **상용 운영 무료:** 사실상 어려움
- **개발/테스트 무료:** 일부 사업자에서 샌드박스/체험 크레딧 가능
- 결론: 예산이 제한적이면, 초기에는 무료 크레딧으로 PoC를 수행하고 운영 전 유료단가 확정이 필요

## 8. 이슈 #5 기준 권장안
1. 1차: 통합 본인인증(PortOne 등)으로 빠른 검증
2. 2차: 트래픽/비용 데이터 확보 후 직접형 전환 여부 결정
3. 전화번호는 소셜 응답만 신뢰하지 말고 OTP 재검증으로 `verified=true` 확정
