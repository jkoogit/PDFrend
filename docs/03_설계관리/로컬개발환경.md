# 08. 로컬 개발 환경 설정 및 온보딩 가이드

## 목적

이 문서는 PDFrend 백엔드 개발자를 위한 **로컬 개발 환경 설정 가이드**입니다. 새로운 개발자가 빠르게 개발 환경을 구성하고, 프로젝트에 기여할 수 있도록 단계별 지침을 제공합니다.

## 2. 단계별 설정 가이드

### 2.1 Step 1: 저장소 클론

```bash
# 1. 저장소 클론
git clone https://github.com/your-org/PDFrend_Project.git
cd PDFrend_Project

# 2. 브랜치 확인
git branch -a
# 출력: dev, stg, main

# 3. dev 브랜치로 전환 (개발은 dev에서 시작)
git checkout dev

# 4. 최신 변경사항 가져오기
git pull origin dev
```

### 2.2 Step 2: 백엔드 프로젝트 진입

```bash
# 백엔드 디렉토리로 이동
cd src/backend

# 현재 위치 확인
pwd
# 출력: /path/to/PDFrend_Project/src/backend
```

### 2.3 Step 3: 의존성 설치

```bash
# 1. pnpm으로 의존성 설치
pnpm install

# 2. 설치 확인
pnpm list
# 주요 패키지 확인:
# - express@4.x.x
# - trpc@11.x.x
# - drizzle-orm@latest
# - postgres@latest
# - typescript@latest
# - vitest@latest
```

### 2.4 Step 4: PostgreSQL 설정

**옵션 A: Docker 사용 (권장)**

```bash
# 1. Docker 이미지 실행
docker run --name pdfrend-dev-postgres \
  -e POSTGRES_USER=pdfrend_dev \
  -e POSTGRES_PASSWORD=local_dev_password \
  -e POSTGRES_DB=pdfrendd \
  -p 5432:5432 \
  -d postgres:16-alpine

# 2. 컨테이너 상태 확인
docker ps | grep pdfrend-dev-postgres

# 3. 컨테이너 로그 확인
docker logs pdfrend-dev-postgres

# 4. 컨테이너 중지 (필요시)
docker stop pdfrend-dev-postgres

# 5. 컨테이너 재시작
docker start pdfrend-dev-postgres
```

**옵션 B: 로컬 PostgreSQL 설치**

```bash
# macOS (Homebrew)
brew install postgresql@16
brew services start postgresql@16

# Ubuntu/Debian
sudo apt-get install postgresql-16
sudo systemctl start postgresql

# 데이터베이스 및 사용자 생성
psql -U postgres -c "CREATE USER pdfrend_dev WITH PASSWORD 'local_dev_password';"
psql -U postgres -c "CREATE DATABASE pdfrendd OWNER pdfrend_dev;"
psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE pdfrendd TO pdfrend_dev;"
```

**연결 테스트:**

```bash
# PostgreSQL 연결 확인
psql -h localhost -U pdfrend_dev -d pdfrendd -c "SELECT version();"

# 예상 출력:
# PostgreSQL 16.13 on x86_64-pc-linux-gnu, compiled by gcc (GCC) 9.4.0, 64-bit
```

### 2.5 Step 5: 환경 변수 설정

```bash
# 1. .env.local 파일 생성
cp .env.example .env.local

# 2. .env.local 편집
nano .env.local

# 3. 다음 값으로 설정:
```

**.env.local 예시:**

```bash
# 환경 식별
NODE_ENV=development
ENVIRONMENT=dev

# 데이터베이스
DB_HOST=localhost
DB_PORT=5432
DB_USER=pdfrend_dev
DB_PASSWORD=local_dev_password
DB_NAME=pdfrendd

# JWT (로컬 개발용 임시값)
JWT_SECRET=local_dev_jwt_secret_do_not_use_in_production_12345

# OAuth (로컬 개발용)
OAUTH_SERVER_URL=https://api.manus.im
VITE_APP_ID=dev-client-id
OAUTH_CLIENT_SECRET=dev-client-secret

# 저장소 (로컬 파일 시스템)
STORAGE_PROVIDER=local
STORAGE_PATH=./uploads
```

### 2.6 Step 6: 데이터베이스 마이그레이션

```bash
# 1. 마이그레이션 SQL 생성 (스키마 변경 시)
pnpm drizzle-kit generate

# 2. 마이그레이션 실행
pnpm drizzle-kit push

# 3. 마이그레이션 상태 확인
pnpm drizzle-kit status

# 4. 데이터베이스 스키마 확인
psql -h localhost -U pdfrend_dev -d pdfrendd -c "\dt"

# 예상 출력:
#  public | users          | table | pdfrend_dev
#  public | documents      | table | pdfrend_dev
#  public | versions       | table | pdfrend_dev
#  public | annotations    | table | pdfrend_dev
#  public | shares         | table | pdfrend_dev
```

### 2.7 Step 7: 서버 시작

```bash
# 1. 개발 서버 시작
pnpm dev

# 2. 예상 출력:
# [OAuth] Initialized with baseURL: https://api.manus.im
# Server running on http://localhost:3000/

# 3. 서버 상태 확인 (다른 터미널)
curl http://localhost:3000/api/health

# 4. tRPC 엔드포인트 확인
curl http://localhost:3000/api/trpc/auth.me
```

## 4. 개발 워크플로우

### 4.1 기능 개발 절차

**1단계: 이슈 생성**

```bash
# GitHub에서 이슈 생성
# 예: #001 - User 도메인 구현
# 라벨: feature, phase-1-1
```

**2단계: 브랜치 생성**

```bash
# dev 브랜치에서 피처 브랜치 생성
git checkout dev
git pull origin dev
git checkout -b feat/#001-user-domain

# 브랜치명 규칙: feat/#<issue-number>-<description>
```

**3단계: 코드 작성**

```bash
# 예: User 도메인 엔티티 작성
# server/domains/user/entities/User.ts

# 예: User 도메인 서비스 작성
# server/domains/user/services/UserDomainService.ts

# 예: User 저장소 구현
# server/infrastructure/persistence/UserRepositoryImpl.ts

# 예: User 애플리케이션 서비스 작성
# server/applications/user/UserApplicationService.ts

# 예: User tRPC 라우터 작성
# server/routers/user.ts
```

**4단계: 테스트 작성**

```bash
# 유닛 테스트 작성
# server/domains/user/services/UserDomainService.test.ts

# 테스트 실행
pnpm test

# 테스트 커버리지 확인
pnpm test:coverage
```

**5단계: 코드 리뷰**

```bash
# 커밋 (jkman 이름으로)
git config user.name "jkman"
git config user.email "jkman@manus.im"
git add .
git commit -m "feat: User 도메인 구현 (#001)"

# 푸시
git push origin feat/#001-user-domain

# Pull Request 생성
# dev 브랜치로 머지 요청
```

**6단계: 머지**

```bash
# PR 승인 후 머지
# dev 브랜치에 통합

# 로컬에서 dev 업데이트
git checkout dev
git pull origin dev
```

### 4.2 Git 커밋 규칙

**커밋 메시지 형식:**

```
<type>: <subject> (#<issue-number>)

<body>

<footer>
```

**타입 분류:**

| 타입 | 설명 | 예시 |
| :---: | :--- | :--- |
| `feat` | 새로운 기능 | `feat: User 도메인 구현 (#001)` |
| `fix` | 버그 수정 | `fix: 문서 업로드 오류 (#002)` |
| `refactor` | 코드 리팩토링 | `refactor: User 서비스 최적화 (#003)` |
| `test` | 테스트 추가 | `test: User 도메인 테스트 (#001)` |
| `docs` | 문서 작성 | `docs: User 도메인 설계 문서 (#001)` |
| `chore` | 빌드, 의존성 | `chore: 의존성 업그레이드 (#004)` |

**예시:**

```bash
git commit -m "feat: User 도메인 구현 (#001)

- User 엔티티 및 값 객체 정의
- UserDomainService 구현
- UserRepositoryImpl 구현
- 유닛 테스트 작성 (95% 커버리지)

Closes #001"
```

## 5. 개발 중 자주 사용하는 명령어

### 5.1 개발 서버 관리

```bash
# 개발 서버 시작
pnpm dev

# 개발 서버 종료
# Ctrl + C

# 개발 서버 재시작 (의존성 변경 시)
pnpm dev:restart
```

### 5.2 테스트 실행

```bash
# 모든 테스트 실행
pnpm test

# 특정 파일 테스트
pnpm test server/domains/user/services/UserDomainService.test.ts

# 테스트 감시 모드 (파일 변경 시 자동 실행)
pnpm test:watch

# 테스트 커버리지 확인
pnpm test:coverage
```

### 5.3 코드 품질 확인

```bash
# ESLint 실행
pnpm lint

pnpm lint:fix

# TypeScript 타입 체크
pnpm typecheck

# 전체 코드 품질 확인
pnpm check
```

### 5.4 데이터베이스 관리


## 7. 문제 해결 (Troubleshooting)

### 7.1 PostgreSQL 연결 오류

**오류 메시지:**

```
Error: connect ECONNREFUSED 127.0.0.1:5432
```

**해결 방법:**

```bash
# 1. PostgreSQL 서비스 상태 확인
docker ps | grep pdfrend-dev-postgres

# 2. 컨테이너가 없으면 다시 생성
docker run --name pdfrend-dev-postgres \
  -e POSTGRES_USER=pdfrend_dev \
  -e POSTGRES_PASSWORD=local_dev_password \
  -e POSTGRES_DB=pdfrendd \
  -p 5432:5432 \
  -d postgres:16-alpine

# 3. 연결 테스트
psql -h localhost -U pdfrend_dev -d pdfrendd -c "SELECT 1;"
```

### 7.2 의존성 설치 오류

**오류 메시지:**

```
npm ERR! code ERESOLVE
npm ERR! ERESOLVE unable to resolve dependency tree
```

**해결 방법:**

```bash
# 1. node_modules 및 lock 파일 삭제
rm -rf node_modules pnpm-lock.yaml

# 2. 의존성 재설치
pnpm install

# 3. 여전히 오류면 캐시 초기화
pnpm store prune
pnpm install
```

### 7.3 포트 이미 사용 중

**오류 메시지:**

```
Error: listen EADDRINUSE: address already in use :::3000
```

**해결 방법:**

```bash
# 1. 포트 3000을 사용하는 프로세스 확인
lsof -i :3000

# 2. 프로세스 종료
kill -9 <PID>

# 또는 다른 포트 사용
PORT=3001 pnpm dev
```

### 7.4 환경 변수 오류

**오류 메시지:**

```
Error: Missing required environment variable: DB_PASSWORD
```

**해결 방법:**

```bash
# 1. .env.local 파일 확인
cat .env.local

# 2. 필수 변수 확인
# DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME 등

# 3. 파일 수정 후 서버 재시작
pnpm dev
```
