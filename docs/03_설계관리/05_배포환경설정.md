# 05. 배포 환경 설정 (Deployment Environment)

## 개요
PDFrend 프로젝트는 Cloudtype을 메인 배포 플랫폼으로 사용하며, GitHub Actions를 통해 지속적 통합 및 배포(CI/CD)를 수행합니다.

## 배포 아키텍처
1. **Source**: GitHub (`dev`, `stg`, `main` 브랜치)
2. **CI/CD**: GitHub Actions
3. **Platform**: Cloudtype (Node.js 런타임)

## GitHub Actions 설정
`.github/workflows/deploy-dev.yml` 파일을 통해 자동 배포가 관리됩니다.

### 주요 설정 항목
- **Trigger**: `dev` 브랜치에 푸시 발생 시 실행
- **Context Root**: `src/backend` (백엔드 소스 경로)
- **Secrets**:
  - `CLOUDTYPE_TOKEN`: Cloudtype API 토큰
  - `GHP_TOKEN`: GitHub Personal Access Token (workflow 권한 필요)

## Cloudtype 설정
- **App**: `node@20`
- **Port**: `4000`
- **Install Command**: `npm install`
- **Build Command**: `npm run build`
- **Start Command**: `npm start`

## 환경 변수 관리
민감 정보는 Cloudtype 대시보드의 환경 변수 설정을 통해 관리하며, 상세 항목은 [환경변수관리대장](../01_정책관리/03_환경변수관리대장.md)을 참조합니다.

## 배포 프로세스
1. `dev` 브랜치 작업 완료 및 푸시
2. GitHub Actions 자동 빌드 및 Cloudtype 배포 시작
3. 배포 완료 후 헬스체크 API(`https://.../health`) 검증
4. 검증 완료 시 `stg`, `main` 브랜치로 승격(Promotion)

---

## 기존 설정 관리 전략 (참조)

### 1. 설정 관리 원칙
PDFrend의 설정 관리는 다음 원칙을 따릅니다:
1. **민감정보 분리**: 암호, API 키, 토큰은 설정 파일에 저장하지 않음
2. **환경 변수 사용**: 배포 시스템(Cloudtype, Docker, 로컬)의 환경 변수로 주입
3. **공통 설정 코드화**: 비즈니스 로직 관련 설정은 코드에 포함

### 2. 설정 파일 구조
```
backend/
├── config/
│   ├── index.ts                    # 설정 진입점
│   ├── common.ts                   # 공통 설정
│   ├── dev.ts                      # 개발 환경 설정
│   ├── stg.ts                      # 검증 환경 설정
│   └── prd.ts                      # 운영 환경 설정
```
