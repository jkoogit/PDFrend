# 07. 배포 환경별 설정 관리 전략

## 목적

이 문서는 PDFrend의 **배포 환경별 설정 관리 전략**을 정의합니다. 개발(dev), 검증(stg), 운영(prd) 환경에서 **공통 설정**과 **환경별 민감정보**를 명확히 분리하고, 보안과 유지보수성을 보장합니다.

---

## 1. 설정 관리 원칙

### 1.1 핵심 원칙

PDFrend의 설정 관리는 다음 원칙을 따릅니다:

1. **민감정보 분리**: 암호, API 키, 토큰은 설정 파일에 저장하지 않음
2. **환경 변수 사용**: 배포 시스템(Cloudtype, Docker, 로컬)의 환경 변수로 주입
3. **공통 설정 코드화**: 비즈니스 로직 관련 설정은 코드에 포함
4. **감사 추적**: 모든 설정 변경 사항 기록
5. **버전 관리**: 공통 설정은 Git에서 관리, 민감정보는 제외

### 1.2 설정 분류

| 분류 | 저장 위치 | 관리 방식 | 예시 |
| :---: | :--- | :--- | :--- |
| **공통 설정** | `config/common.ts` | Git 관리 | API 버전, 페이지 크기, 타임아웃 |
| **환경별 설정** | `config/dev.ts`, `config/stg.ts`, `config/prd.ts` | Git 관리 | DB 호스트, 로그 레벨, 기능 플래그 |
| **민감정보** | 환경 변수 | 배포 시스템 주입 | DB 암호, API 키, JWT 시크릿 |
| **로컬 개발** | `.env.local` | Git 제외 (.gitignore) | 로컬 DB 연결, 개발 API 키 |

## 4. 설정 로더 구현

### 4.1 설정 진입점 (config/index.ts)

```typescript
// config/index.ts
import { commonConfig } from './common';
import { devConfig } from './dev';
import { stgConfig } from './stg';
import { prdConfig } from './prd';
import { validateConfig } from './validation';

type Environment = 'dev' | 'stg' | 'prd';

function getEnvironment(): Environment {
  const env = process.env.ENVIRONMENT || process.env.NODE_ENV;
  
  if (env === 'production') return 'prd';
  if (env === 'staging' || env === 'stg') return 'stg';
  return 'dev';
}

function loadConfig() {
  const environment = getEnvironment();
  
  let envConfig;
  switch (environment) {
    case 'prd':
      envConfig = prdConfig;
      break;
    case 'stg':
      envConfig = stgConfig;
      break;
    case 'dev':
    default:
      envConfig = devConfig;
      break;
  }

  // 공통 설정과 환경별 설정 병합
  const config = {
    ...commonConfig,
    ...envConfig,
  };

  // 설정 검증
  validateConfig(config);

  return config;
}

export const config = loadConfig();
```

### 4.2 환경 변수 로더 (server/_core/env.ts)

```typescript
// server/_core/env.ts
import { z } from 'zod';

const envSchema = z.object({
  // 필수 환경 변수
  NODE_ENV: z.enum(['development', 'production']).default('development'),
  ENVIRONMENT: z.enum(['dev', 'stg', 'prd']).default('dev'),
  
  // 데이터베이스
  DB_HOST: z.string().default('localhost'),
  DB_PORT: z.coerce.number().default(5432),
  DB_USER: z.string(),
  DB_PASSWORD: z.string(),
  DB_NAME: z.string(),
  
  // JWT
  JWT_SECRET: z.string(),
  
  // OAuth
  OAUTH_SERVER_URL: z.string().url(),
  VITE_APP_ID: z.string(),
  OAUTH_CLIENT_SECRET: z.string(),
  
  // 저장소 (선택)
  S3_BUCKET: z.string().optional(),
  S3_REGION: z.string().optional(),
  S3_ACCESS_KEY_ID: z.string().optional(),
  S3_SECRET_ACCESS_KEY: z.string().optional(),
});

export type Env = z.infer<typeof envSchema>;

export function loadEnv(): Env {
  const result = envSchema.safeParse(process.env);
  
  if (!result.success) {
    console.error('❌ 환경 변수 검증 실패:');
    result.error.errors.forEach(error => {
      console.error(`  - ${error.path.join('.')}: ${error.message}`);
    });
    process.exit(1);
  }
  
  return result.data;
}

export const env = loadEnv();
```

## 6. 설정 검증 및 모니터링

### 6.1 설정 검증 (config/validation.ts)

```typescript
// config/validation.ts
import { z } from 'zod';

const configSchema = z.object({
  environment: z.enum(['development', 'staging', 'production']),
  nodeEnv: z.enum(['development', 'production']),
  database: z.object({
    host: z.string().min(1),
    port: z.number().min(1).max(65535),
    database: z.string().min(1),
    user: z.string().min(1),
    password: z.string().min(1),
  }),
  oauth: z.object({
    baseUrl: z.string().url(),
    clientId: z.string().min(1),
    clientSecret: z.string().min(1),
  }),
  storage: z.object({
    provider: z.enum(['local', 's3']),
  }),
});

export function validateConfig(config: any) {
  const result = configSchema.safeParse(config);
  
  if (!result.success) {
    console.error('❌ 설정 검증 실패:');
    result.error.errors.forEach(error => {
      console.error(`  - ${error.path.join('.')}: ${error.message}`);
    });
    process.exit(1);
  }
  
  console.log('✅ 설정 검증 성공');
  console.log(`   환경: ${config.environment}`);
  console.log(`   데이터베이스: ${config.database.host}:${config.database.port}/${config.database.database}`);
  console.log(`   저장소: ${config.storage.provider}`);
}
```
