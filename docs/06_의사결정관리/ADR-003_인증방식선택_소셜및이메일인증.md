# ADR-003: 사용자 인증 방식 선택 (소셜 및 이메일 인증)

## 1. 상태
**승인됨 (Approved)**

## 2. 배경
초기에는 Manus OAuth를 기본 인증 방식으로 고려하였으나, 서비스의 범용성과 사용자 접근성을 높이기 위해 국내외 주요 소셜 로그인 및 전통적인 이메일 인증 방식을 도입하기로 결정하였습니다.

## 3. 결정 사항
다음과 같은 다중 인증 체계를 구축하기로 결정하였습니다.
*   **소셜 로그인**: 카카오톡, 네이버, 구글 OAuth 2.0 연동
*   **자체 인증**: 이메일 및 비밀번호 기반 인증
*   **기술 스택**: Passport.js를 활용한 전략(Strategy) 패턴 적용

## 4. 사유
1.  **사용자 편의성**: 국내 사용자가 가장 많이 사용하는 카카오, 네이버와 글로벌 표준인 구글 로그인을 제공하여 가입 허들을 낮춥니다.
2.  **범용성**: 특정 플랫폼(Manus)에 종속되지 않고 독립적인 서비스 운영이 가능합니다.
3.  **백업 수단**: 소셜 계정이 없는 사용자를 위해 이메일 인증 방식을 병행하여 사용자층을 확대합니다.
4.  **확장성**: Passport.js의 전략 패턴을 사용하면 향후 새로운 인증 수단(예: Apple ID)을 추가할 때 기존 코드의 변경을 최소화할 수 있습니다.

## 5. 결과 및 영향
*   **긍정적**: 사용자 선택권 확대, 서비스 독립성 확보.
*   **부정적**: 각 소셜 플랫폼별 API 키 관리 및 개발 공수 증가, 이메일 인증을 위한 메일 서버(SMTP) 연동 필요.
