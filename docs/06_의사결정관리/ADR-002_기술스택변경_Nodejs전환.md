# ADR-002: 기술 스택 변경 (Kotlin/Spring Boot -> Node.js/Express/tRPC)

## 1. 상태
**승인됨 (Approved)**

## 2. 배경
초기 프로젝트 계획에서는 Kotlin과 Spring Boot(코프링)를 백엔드 기술 스택으로 고려하였습니다. 그러나 프로젝트의 목표인 '빠른 MVP 개발'과 '학습 효율성', 그리고 'Manus WebDev 환경과의 최적의 호환성'을 고려할 때 기존 스택의 유지 여부를 재검토하게 되었습니다.

## 3. 결정 사항
백엔드 기술 스택을 다음과 같이 변경하기로 결정하였습니다.
*   **런타임**: Node.js (v22.13.0)
*   **언어**: TypeScript
*   **프레임워크**: Express.js + tRPC
*   **ORM**: Drizzle ORM (PostgreSQL)

## 4. 사유
1.  **개발 생산성**: TypeScript와 tRPC의 조합은 클라이언트-서버 간 타입 공유를 통해 개발 속도를 획기적으로 높여줍니다.
2.  **환경 최적화**: Manus WebDev 플랫폼의 기본 스캐폴딩이 Node.js 기반으로 구성되어 있어, 인프라 설정 및 배포(Cloudtype) 과정에서의 마찰을 최소화할 수 있습니다.
3.  **학습 곡선**: 팀 내 JavaScript/TypeScript 경험을 즉시 활용할 수 있어, 비즈니스 로직 구현에 더 집중할 수 있습니다.
4.  **비용 효율성**: 상대적으로 가벼운 Node.js 런타임은 NAS 또는 저사양 우분투 환경에서도 효율적으로 동작합니다.

## 5. 결과 및 영향
*   **긍정적**: 타입 안정성 확보, API 명세서 자동화 효과(tRPC), 빠른 배포 사이클.
*   **부정적**: 기존에 작성된 코프링 관련 설계 문서의 현행화 작업 필요.
