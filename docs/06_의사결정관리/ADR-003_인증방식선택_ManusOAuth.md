# ADR-003: 사용자 인증 방식 선택 (Manus OAuth)

## 1. 상태
**승인됨 (Approved)**

## 2. 배경
Phase 1의 핵심 기능인 '사용자 인증 및 권한 관리'를 구현하기 위해, 자체 Passport 기반 인증 시스템과 Manus 플랫폼에서 제공하는 OAuth 시스템 중 하나를 선택해야 합니다.

## 3. 결정 사항
기본 인증 방식으로 **Manus OAuth**를 채택하고, 확장성을 위해 **Passport.js** 구조를 내부적으로 유지하기로 결정하였습니다.

## 4. 사유
1.  **통합 편의성**: Manus 플랫폼 내에서 동작하는 서비스로서, 별도의 회원가입 절차 없이 Manus 계정으로 즉시 연동이 가능합니다.
2.  **보안성**: 민감한 사용자 비밀번호를 직접 관리하지 않고 신뢰할 수 있는 OAuth 제공자에게 위임함으로써 보안 리스크를 줄입니다.
3.  **개발 속도**: 복잡한 인증 로직(로그인, 세션, 토큰 갱신 등)을 직접 구현하는 대신 제공된 API를 활용하여 핵심 비즈니스 로직에 집중할 수 있습니다.
4.  **유연성**: 향후 다른 OAuth 제공자(Google, GitHub 등)를 추가할 때를 대비하여 Passport.js 전략(Strategy) 패턴을 적용하여 구조적 유연성을 확보합니다.

## 5. 결과 및 영향
*   **긍정적**: 인증 구현 시간 단축, 사용자 접근성 향상.
*   **부정적**: Manus 플랫폼 의존성 발생 (단, Passport 전략 패턴으로 완화 가능).
