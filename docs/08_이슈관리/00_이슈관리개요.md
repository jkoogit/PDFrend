# 08. 이슈 관리 (Issue Management)

## 목적
버그, 개선 사항, 논의 필요 항목 등 프로젝트 진행 중 발생하는 모든 이슈를 추적하고 관리합니다.

## 이슈 관리 규칙
모든 이슈 리포트는 다음의 파일명 규칙을 준수하여 작성합니다.

### 파일명 규칙
`{YYMMDD}_{순번}_{문제}_{원인}.md`
- 예시: `260513_01_cloudtype배포이슈-git폴더서브모듈.md`

### 작성 원칙
1. **문제 현상(Problem)**: 발생한 현상을 구체적으로 기술
2. **원인 분석(Root Cause)**: 기술적/프로세스적 원인 파악
3. **해결 방법(Solution)**: 적용된 조치 사항 기록
4. **재발 방지(Prevention)**: 향후 동일 이슈 방지를 위한 대책 수립
