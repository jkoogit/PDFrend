# 08. 이슈 관리 (Issue Management)

## 목적
버그, 개선 사항, 논의 필요 항목 등 프로젝트 진행 중 발생하는 모든 이슈를 추적하고 관리합니다.

## 운영 원칙 (Git 전략 연계)
- 모든 작업은 **이슈 등록 후 시작**한다.
- 작업 브랜치는 원칙적으로 `main` 기준 `feature/#이슈번호-작업명`으로 생성한다.
- 저장소에 `main` 로컬 브랜치가 없을 경우, 기준 브랜치 부재 사유를 이슈 문서에 먼저 기록한 뒤 대체 기준 브랜치에서 시작한다.
