## 🔎 이슈 #5 맞춤 구현 범위
- 간편인증 제공자 연동: 카카오/네이버/구글
- 내부 식별자: `provider_cd + provider_key_val` 기반 인증수단 식별
- 이메일: 로그인 필수조건이 아닌 협업/공지 채널로 분리
- 계정연결: 로그인 세션 기반 명시적 link/unlink만 허용
- 자동 병합: 금지(충돌 시 수동 검증 플로우)

## 🧪 검증 포인트
- state/nonce 검증 실패 시 인증 차단
- 동일 `provider_cd + provider_key_val` 중복 연결 방지
- 이메일 미인증 상태에서도 간편로그인 가능
- 인증수단 삭제 시 최소 1개 로그인 수단 유지 정책 검토
