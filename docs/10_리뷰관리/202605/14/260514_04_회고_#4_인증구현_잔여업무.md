# 260514_04_회고_#4_인증구현_잔여업무

## 1. 목적
이슈 #4 인증 구현 진행 중, 즉시 완료 불가한 항목을 잔여업무로 명확히 기록하여 후속 스프린트에서 누락 없이 처리한다.

## 2. 잔여업무 목록
1. Role/권한 모델 연계
   - 내용: 사용자 Role(예: admin/editor/user) 모델 정의 및 CRUD 접근정책 연결
   - 선행조건: 권한 정책 합의(리소스 단위 권한표)
   - 목표 단계: 실행계획 단계 B(P1)

2. 토큰 회전/재발급 정책
   - 내용: access token 만료 시 재발급 플로우(refresh token 포함 여부) 확정
   - 선행조건: 보안 기본선(토큰 수명/보관위치) 확정
   - 목표 단계: 실행계획 단계 A(P0)~C(P1)

3. 블랙리스트 영속화
   - 내용: 현재 메모리 denylist를 Redis/DB로 전환
   - 선행조건: 운영환경 저장소 선택 및 TTL 전략 확정
   - 목표 단계: 실행계획 단계 D(P2)

4. 실패 시나리오 자동화 테스트
   - 내용: 위변조 토큰/만료 토큰/중복가입/잘못된 비밀번호 테스트 자동화
   - 선행조건: 테스트 러너 및 fixture 전략 고정
   - 목표 단계: 실행계획 단계 C(P1)

## 3. 결정 연계
- 이슈 번호는 `#4=인증 구현`으로 확정한다.
- 현재 토큰 방식은 HMAC 유지, JWT 전환은 잔여업무 검토 항목으로 유지한다.
- 로그아웃 정책은 서버측 블랙리스트(현재 메모리 denylist) 유지, 후속 영속화 진행.
