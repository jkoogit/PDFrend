# 260513_03_이슈별_작업범위_점검_보고서

## 1. 개요
본 보고서는 PDFrend 프로젝트의 전체 로드맵 및 프로덕트 요구사항(PRD) 대비 현재까지 진행된 GitHub 이슈들의 작업 범위를 점검하고, 누락된 핵심 기능을 도출하여 향후 작업 계획 수립에 기여하는 것을 목적으로 합니다. 또한, 작업 내용에 식별 번호를 부여하여 추적성을 높입니다.

## 2. 현재까지 진행된 GitHub 이슈 요약
현재까지 PDFrend 프로젝트에서 완료 또는 진행 중인 GitHub 이슈는 다음과 같습니다.

| 이슈 ID | 제목 | 상태 |
|---|---|---|
| #1 | 백엔드 프로젝트 초기 설정 및 환경 구축 | 완료 |
| #2 | DB 스키마 설계 및 Drizzle ORM 설정 | 완료 |
| #3 | User 및 Document 도메인 CRUD API 구현 | 진행 중 (PR 생성 완료) |

## 3. 로드맵 및 요구사항 분석

### 3.1. 로드맵 (Phase 1: 코어 백엔드 구축)
[docs/07_일정관리/01_로드맵관리.md]에 따르면, Phase 1의 목표는 "사용자 인증, DB 스키마, 파일 업로드/다운로드, API 기본 골격 완성"입니다. 세부 스프린트 작업 내용은 다음과 같습니다.

*   **Sprint 1-1**: DB 스키마 설계 및 마이그레이션 (users, documents, versions, annotations, bookmarks 테이블, PostgreSQL JSONB, Drizzle ORM)
*   **Sprint 1-2**: 사용자 인증 및 권한 관리 API (OAuth 인증 흐름, 사용자 역할 기반 권한 체계, 사용자별 용량 할당 API)
*   **Sprint 1-3**: 문서 업로드 및 리소스 분리 (PDF 파일 업로드 API, 페이지별 이미지 추출 및 저장, 메타데이터 기록)
*   **Sprint 1-4**: 문서 다운로드 및 병합 (이미지 + OCR + 주석 병합 엔진, 파일명 포맷 생성, Xodo 호환 표준 Annotation 매핑)

Phase 1의 핵심 산출물은 "문서 CRUD API, 파일 업로드/다운로드 API, DB 마이그레이션 스크립트"입니다.

### 3.2. 프로덕트 요구사항 정의서 (PRD)
[docs/02_기획관리/01_프로덕트요구사항.md]의 "2. 상세 기능 명세" 섹션은 다음과 같은 핵심 기능을 정의하고 있습니다.

*   **2.1 문서 라이프사이클 관리**:
    *   용량 관리 (Policy-STG-001, Policy-STG-002, Policy-STG-003)
    *   온라인 업로드 (파일명 기반 도서 판별, 이미지/OCR/주석 분리 저장, 메타데이터 기록)
    *   다운로드 (리소스 병합, 파일명 포맷)
*   **2.2 버전 및 동시성 관리 정책**:
    *   버전 관리 모델 (Git 방식 브랜치/머지)
    *   실시간 저장 및 체크포인트
    *   동시 작업자 처리
*   **2.3 공유 및 보안 정책**:
    *   공유 링크 속성 (대상, 기간, 권한)
    *   실시간 권한 반영

## 4. 이슈별 작업 범위 점검 및 번호 부여

### 4.1. 이슈 #1: 백엔드 프로젝트 초기 설정 및 환경 구축
*   **작업 내용**: 샌드박스 환경 설정, GitHub 저장소 클론, pnpm 모노레포 구조 확인, Cloudtype 배포 환경 구성.
*   **작업 번호**: 1.1.1 ~ 1.1.4
*   **로드맵/PRD 부합**: Phase 0의 "프로젝트 기반 구축" 및 Phase 1의 "API 기본 골격 완성"을 위한 선행 작업으로 적절합니다.

### 4.2. 이슈 #2: DB 스키마 설계 및 Drizzle ORM 설정
*   **작업 내용**: PostgreSQL 도입, Drizzle ORM 설정, User 및 Document 테이블 스키마 설계 및 반영.
*   **작업 번호**: 2.1.1 ~ 2.1.4
*   **로드맵/PRD 부합**: Phase 1의 "Sprint 1-1: DB 스키마 설계 및 마이그레이션"에 직접적으로 부합하며, "API 기본 골격 완성"의 핵심 요소입니다.

### 4.3. 이슈 #3: User 및 Document 도메인 CRUD API 구현
*   **작업 내용**: User 및 Document 도메인에 대한 CRUD API 구현 (tRPC), Drizzle ORM 연동, 권한 체크 로직 포함, ESLint 설정 최적화, 테스트 계획 및 결과 보고서 작성.
*   **작업 번호**: 3.1.1 ~ 3.1.6
*   **로드맵/PRD 부합**: Phase 1의 핵심 산출물인 "문서 CRUD API"에 직접적으로 부합하며, PRD의 "2.1 문서 라이프사이클 관리"의 기초를 마련합니다.

## 6. 결론 및 향후 계획
현재까지의 이슈 진행은 Phase 1의 핵심 목표 중 "DB 스키마" 및 "문서 CRUD API"의 기반을 성공적으로 마련했습니다. 하지만 "사용자 인증", "파일 업로드/다운로드"와 관련된 핵심 기능들이 아직 이슈로 정의되지 않았습니다.

**향후 계획**:
1.  제안된 이슈 #4, #5, #6을 GitHub에 등록하여 Phase 1의 모든 핵심 기능을 포괄하도록 합니다.
2.  각 이슈에 대해 상세한 작업 계획을 수립하고, 문서화 지침에 따라 테스트 계획 및 결과 보고서를 작성합니다.
3.  모든 작업 내용에 식별 번호를 부여하여 추적성을 높이고, 진행 상황을 명확히 관리합니다.
