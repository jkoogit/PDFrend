# PDFrend 프로젝트 문서 가이드

## 📚 문서 구조 개요

PDFrend 프로젝트의 모든 문서는 **8개의 카테고리**로 체계적으로 관리됩니다. 각 카테고리는 프로젝트 개발의 특정 측면을 담당하며, 상호 연관되어 있습니다.

```
docs/
├── 01_정책관리/              # 시스템 운영 정책
├── 02_기획관리/              # 기획 및 요구사항
├── 03_설계관리/              # 아키텍처 및 설계
├── 04_기술관리/              # 기술 스택 및 구현
├── 05_테스트관리/            # 테스트 전략 및 케이스
├── 06_의사결정관리/          # 아키텍처 결정 기록
├── 07_일정관리/              # 프로젝트 일정
├── 08_이슈관리/              # 이슈 템플릿 및 체크리스트
└── README.md                 # 이 파일
```

---

## 🎯 카테고리별 가이드

### 01_정책관리 (System Policy Management)

**목적:** 시스템 운영에 필요한 모든 정책과 규칙을 정의합니다.

**주요 문서:**
- `시스템정책.md` - 전체 시스템 정책 목록
- `사용자정책.md` - 사용자 관련 정책
- `문서정책.md` - 문서 관련 정책
- `공유정책.md` - 공유 및 접근 제어 정책
- `보안정책.md` - 보안 및 개인정보 정책

**언제 참고하나요?**
- 새로운 기능 개발 시 관련 정책 확인
- 정책 변경이 필요할 때
- 규정 준수 검증 시

---

### 02_기획관리 (Planning Management)

**목적:** 프로젝트 기획, 요구사항, 사용자 스토리를 관리합니다.

**주요 문서:**
- `프로젝트개요.md` - 프로젝트 비전, 목표, 범위
- `요구사항정의.md` - 기능 요구사항 (FRD)
- `사용자스토리.md` - 사용자 스토리 및 페르소나
- `프로덕트로드맵.md` - 릴리스 계획 및 마일스톤
- `화면정의/` - UI/UX 설계 문서
- `프로세스정의/` - 비즈니스 프로세스 정의

**언제 참고하나요?**
- 새로운 기능 개발 시 요구사항 확인
- 사용자 관점에서 기능 이해
- 프로덕트 로드맵 확인

---

### 03_설계관리 (Architecture & Design Management)

**목적:** 시스템 아키텍처, 데이터 모델, 설계 의도를 관리합니다.

**주요 문서:**
- `아키텍처개요.md` - 아키텍처 최종 요약
- `DDD도메인설계.md` - 8개 도메인 상세 설계
- `다중DB전략.md` - 환경별 DB 관리 전략
- `배포환경설정.md` - 배포 환경 및 설정
- `로컬개발환경.md` - 로컬 개발 환경 설정
- `설계의도.md` - 아키텍처 결정 이유

**언제 참고하나요?**
- 새로운 도메인 추가 시
- 데이터베이스 스키마 변경 시
- 배포 환경 설정 시
- 아키텍처 이해가 필요할 때

---

**목적:** 기술 스택, 구현 가이드, 코드 표준을 관리합니다.

**주요 문서:**
- `기술스택.md` - 사용 기술 및 버전 정보
- `코드표준/` - 코딩 규칙 및 컨벤션

  - `네이밍규칙.md`
  - `Git커밋규칙.md`
- `구현가이드/` - 기능별 구현 방법
  - `도메인구현가이드.md`
  - `애플리케이션서비스구현.md`
  - `tRPC라우터구현.md`
  - `데이터베이스쿼리.md`
  - `에러처리가이드.md`
- `성능최적화.md` - 성능 최적화 전략
- `보안구현.md` - 보안 구현 가이드
- `학습자료/` - 기술 학습 자료

**언제 참고하나요?**
- 코드 작성 시 규칙 확인
- 새로운 기능 구현 방법 학습
- 성능 최적화 필요 시
- 보안 구현 방법 확인

  - `User도메인테스트.md`
  - `Document도메인테스트.md`

  - `Annotation도메인테스트.md`

- `E2E테스트/` - End-to-End 테스트

## 📝 문서 작성 규칙

### 파일명 규칙

**한글 파일명 사용:**
- ✅ `사용자정책.md`
- ❌ `User Policy.md`
- ❌ `사용자 정책.md` (띄어쓰기 금지)

**명사형 사용:**
- ✅ `도메인설계.md`
- ❌ `도메인설계하기.md`

**버전 관리:**
- 중요한 문서는 버전 번호 표시
- 예: `프로젝트개요_v1.0.md`

### 문서 구조

모든 문서는 다음 구조를 따릅니다:

```markdown
# 제목

## 📌 개요
문서의 목적과 범위

## 🎯 목표
달성하려는 목표

## 📚 주요 내용
주요 섹션들...

## ✅ 체크리스트
완료 항목 확인

## 📞 문의
연락처 정보

---

**작성자:** jkman
**작성일:** 2026-05-12
**버전:** 1.0.0
```

**표 사용:**
```markdown
| 항목 | 설명 |
| :---: | :--- |
| 항목1 | 설명1 |
```

**코드 블록:**
```typescript

```

**강조:**

- `코드` - 코드 참조
- > 인용문 - 중요한 문구
