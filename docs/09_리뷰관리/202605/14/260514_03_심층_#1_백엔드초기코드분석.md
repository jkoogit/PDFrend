## 1. 개요

## 2. 핵심 코드 분석

### 2.1 [src/index.ts](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/index.ts) (서버 진입점)
```typescript
import express from 'express';
import * as trpcExpress from '@trpc/server/adapters/express';
import { appRouter } from './api/trpc/router';
import { createContext } from './api/trpc/context';

const app = express(); // (1) Express 앱 인스턴스 생성
const port = process.env.PORT || 4000; // (2) 포트 설정

// (3) tRPC 미들웨어 연결
app.use(
  '/trpc',
  trpcExpress.createExpressMiddleware({
    router: appRouter,
    createContext,
  })
);

// (4) 서버 실행
app.listen(port, () => {
  console.log(`Server is running on port ${port}`);
});
```
- **(1) [Line 7]**: 가장 대중적인 Node.js 웹 프레임워크인 Express를 초기화합니다. ([코드 이동](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/index.ts#L7))
- **(2) [Line 8]**: 환경 변수(`process.env.PORT`)가 있으면 사용하고, 없으면 기본값으로 `4000`을 사용합니다. 이는 로컬 개발과 클라우드 배포 환경 모두에 대응하기 위한 표준 패턴입니다. ([코드 이동](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/index.ts#L8))
- **(3) [Line 11]**: `/trpc` 경로로 들어오는 모든 요청을 tRPC가 처리하도록 어댑터를 연결합니다. ([코드 이동](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/index.ts#L11))
- **(4) [Line 20]**: `app.listen`은 서버를 실제로 가동시키는 명령어입니다. 지정된 포트를 열고 외부 요청을 기다리는 상태가 됩니다. ([코드 이동](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/index.ts#L20))

### 2.2 [src/api/trpc/router.ts](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/api/trpc/router.ts) (API 정의)
```typescript
import { initTRPC } from '@trpc/server';

const t = initTRPC.create(); // (1) tRPC 초기화

export const appRouter = t.router({
  healthCheck: t.procedure.query(() => { // (2) Query 프로시저 정의
    return {
      status: 'ok',
      message: 'PDFrend API is running',
    };
  }),
});
```
- **(1) [Line 3]**: tRPC의 핵심 객체를 생성합니다. 이를 통해 라우터와 프로시저(Procedure)를 정의할 수 있습니다. ([코드 이동](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/api/trpc/router.ts#L3))
- **(2) [Line 6]**: `query`는 데이터를 가져올 때(GET), `mutation`은 데이터를 변경할 때(POST/PUT/DELETE) 사용합니다. ([코드 이동](https://github.com/jkoogit/PDFrend/blob/main/src/backend/src/api/trpc/router.ts#L6))

## 3. 설계 의도 및 학습 포인트
1. **왜 tRPC인가?**: REST API처럼 URL과 Method를 일일이 정의하지 않아도, 함수 호출 방식으로 API를 사용할 수 있으며 서버의 타입이 클라이언트에 실시간으로 공유됩니다.
2. **CORS의 역할**: 보안을 위해 브라우저가 다른 도메인(포트 포함) 간의 통신을 차단하는 것을 방지합니다. 백엔드에서 허용된 프론트엔드 주소를 명시하여 안전한 통신을 가능하게 합니다.
3. **DDD 구조의 이점**: 현재는 파일이 몇 개 없지만, `domain`과 `infrastructure` 폴더를 미리 만들어둠으로써 향후 비즈니스 로직이 복잡해져도 코드가 섞이지 않게 방어막을 쳤습니다.
