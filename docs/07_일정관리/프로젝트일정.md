# 프로젝트 일정 및 마일스톤

## 📌 개요

PDFrend 프로젝트는 **3개의 Phase**로 나누어 진행됩니다. 각 Phase는 명확한 목표, 마일스톤, 그리고 릴리스 계획을 가지고 있습니다.

---

## 🎯 프로젝트 비전

**목표:** PDF 문서 협업 플랫폼의 MVP 완성 → 기능 확대 → 마이크로서비스 전환

**기간:** 2026년 5월 ~ 2027년 12월 (20개월)

**팀:** 백엔드 1명 (jkman), 프론트엔드 (TBD), 인프라 (TBD)

---

## 📊 Phase별 계획

### Phase 1: 초기 개발 (MVP 완성)

**기간:** 2026년 5월 ~ 2026년 9월 (5개월)

**목표:** 핵심 기능 구현 및 MVP 릴리스

**주요 기능:**
- 사용자 관리 (가입, 로그인, 프로필)
- 문서 업로드 및 관리
- 기본 주석 기능
- 문서 공유 (읽기 전용)
- 버전 관리 (기본)

**마일스톤:**

| 마일스톤 | 날짜 | 목표 |
| :---: | :--- | :--- |
| **M1: 아키텍처 완성** | 2026-05-31 | DDD 설계, DB 스키마, 개발 환경 |
| **M2: User 도메인** | 2026-06-15 | 사용자 관리 기능 완성 |
| **M3: Document 도메인** | 2026-07-15 | 문서 업로드, 관리 기능 |
| **M4: Annotation 도메인** | 2026-08-15 | 주석 기능 완성 |
| **M5: MVP 릴리스** | 2026-09-30 | 베타 버전 출시 |

**릴리스:** v0.1.0 (Beta)

---

### Phase 2: 기능 확대 (정식 릴리스)

**기간:** 2026년 10월 ~ 2027년 6월 (9개월)

**목표:** 기능 확대 및 정식 릴리스

**주요 기능:**
- 고급 주석 기능 (그리기, 마크업)
- 실시간 협업 (Sync 도메인)
- 고급 공유 기능 (권한 관리)
- 검색 기능
- 분석 대시보드
- 모바일 앱 (기본)

**마일스톤:**

| 마일스톤 | 날짜 | 목표 |
| :---: | :--- | :--- |
| **M6: Sync 도메인** | 2026-11-15 | 실시간 협업 기능 |
| **M7: Sharing 도메인** | 2026-12-15 | 고급 공유 기능 |
| **M8: 검색 기능** | 2027-01-31 | 전문 검색 기능 |
| **M9: 분석 기능** | 2027-02-28 | 사용 분석 대시보드 |
| **M10: 정식 릴리스** | 2027-03-31 | v1.0.0 정식 출시 |
| **M11: 모바일 앱** | 2027-06-30 | iOS/Android 앱 출시 |

**릴리스:** v1.0.0 (Stable)

---

### Phase 3: 마이크로서비스 전환

**기간:** 2027년 7월 ~ 2027년 12월 (6개월)

**목표:** 마이크로서비스 아키텍처로 전환

**주요 작업:**
- Kotlin으로 각 도메인 마이크로서비스 구현
- 메시지 큐 도입 (RabbitMQ/Kafka)
- API Gateway 구축
- 분산 트레이싱 (Jaeger)
- 모니터링 및 로깅 (ELK Stack)

**마일스톤:**

| 마일스톤 | 날짜 | 목표 |
| :---: | :--- | :--- |
| **M12: User 마이크로서비스** | 2027-08-31 | Kotlin 마이크로서비스 |
| **M13: Document 마이크로서비스** | 2027-09-30 | Kotlin 마이크로서비스 |
| **M14: 나머지 마이크로서비스** | 2027-10-31 | 모든 도메인 마이크로서비스 |
| **M15: 마이크로서비스 릴리스** | 2027-12-31 | v2.0.0 MSA 버전 |

**릴리스:** v2.0.0 (Microservices)

## 🎯 각 Phase별 성공 기준

### Phase 1 성공 기준

- ✅ 모든 핵심 도메인 구현 완료
- ✅ 단위 테스트 커버리지 95% 이상
- ✅ 통합 테스트 모두 통과
- ✅ E2E 테스트 모두 통과
- ✅ 성능 테스트 통과 (응답 시간 < 500ms)
- ✅ 보안 감사 완료
- ✅ 문서 완성도 100%

### Phase 2 성공 기준

- ✅ 모든 추가 기능 구현 완료
- ✅ 테스트 커버리지 유지 (95% 이상)
- ✅ 성능 개선 (응답 시간 < 300ms)
- ✅ 사용자 피드백 반영
- ✅ 모바일 앱 출시

### Phase 3 성공 기준

- ✅ 모든 도메인 마이크로서비스 전환
- ✅ 메시지 큐 기반 통신 구현
- ✅ 분산 트레이싱 및 모니터링 완성
- ✅ 성능 개선 (응답 시간 < 200ms)
- ✅ 자동 스케일링 구현
