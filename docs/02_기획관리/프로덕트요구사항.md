# [기획서] PDFrend: 스마트 PDF 작업 및 동기화 플랫폼 (v2.0)

## 1. 서비스 개요
**PDFrend**는 스캔된 도서 PDF에 OCR과 목차를 적용하고, 다양한 기기에서 중단 없는 주석 작업 및 버전 관리를 지원하는 **클라우드 기반 PDF 작업 관리 플랫폼**입니다.

### 1.1 서비스 배경 및 목적
*   **작업 연속성 확보**: 태블릿(Xodo), PC, 모바일 등 다양한 기기를 오가며 작업할 때 발생하는 파일 전송의 번거로움과 버전 혼선 해결.
*   **디지털 자산화**: 단순 이미지 PDF를 OCR 텍스트와 구조화된 목차 데이터로 변환하여 검색 및 편집이 가능한 형태로 관리.
*   **협업 환경 제공**: 동일 문서에 대해 여러 사용자가 권한에 따라 공동 작업을 수행하고, 실시간으로 주석을 공유.

---

## 2. 상세 기능 명세 (초안 내용 전수 반영)

본 섹션의 기능들은 [시스템 정책서](../01_정책관리/시스템정책.md)에 정의된 정책들을 기반으로 합니다.


### 2.1 문서 라이프사이클 관리
*   **용량 관리**: [Policy-STG-001], [Policy-STG-002], [Policy-STG-003]에 따라 사용자별 용량 할당 및 공유 문서 용량 제외 정책 적용.

1.  **오프라인 준비**: 사용자가 스캔 이미지(JPG)를 PDF로 변환하고 OCR 및 목차를 적용한 최종 PDF 준비.
2.  **온라인 업로드**: 
    *   파일명을 기준으로 기존 도서 여부 자동 판별 및 카테고리 추천.
    *   업로드 시 서버에서 **이미지(JPG), OCR(JSON), 주석/목차(JSON)**로 분리하여 DB 저장.
3.  **온라인 작업**: 뷰어를 통해 실시간 주석 편집 및 동기화.
4.  **다운로드**: 분리된 리소스를 병합하여 표준 PDF 생성. 파일명 포맷: `[도서명]_[버전]_[수정일자]_[수정시간].pdf`.

### 2.2 버전 및 동시성 관리 정책 (확정)
*   **버전 관리 모델**: [Policy-VER-001] Git 방식의 브랜치/머지 모델 도입.
*   **실시간 저장 및 체크포인트**: [Policy-VER-002] 모든 주석 수정은 실시간 저장되며, 명시적 버전 생성 시에만 버전 번호 부여.
*   **동시 작업자 처리**: [Policy-VER-003] 타 작업자가 버전 생성 시 자동 업데이트 및 작업 연속성 유
### 2.3 공유 및 보안 정책
*   **공유 링크 속성**: 대상, 기간, 권한을 세분화하여 설정.
*   **실시간 권한 반영**: 편집 중 권한 변경 시 즉시 뷰어 모드 전환 및 안내 메시지 표시.

## 3. 사용자 경험 (UX) 상세
*   **작성자 표시**: 주석 끝에 작성자 이름/아바타 라벨 표시 (토글 가능). 마우스 오버 또는 롱프레스 시 상세 정보 노출.
*   **검색 기능**: 작성자별, 주석 구분별, 텍스트 검색 지원 (페이지 내/문서 내 범위 설정 가능).
*   **북마크 포맷**: `{페이지}*{날짜}*{작성자}` 등 사용자 정의 포맷 지원.
*   **멀티 문서 탭**: 여러 문서를 동시에 열어 작업 가능하며, 탭 전환 시 각 문서의 마지막 위치 기억.

---

## 4. 기술 요구사항
*   **Frontend**: React 19 + Tailwind CSS 4 + shadcn/ui, PDF.js 커스텀 뷰어, IndexedDB(오프라인 저장).
*   **Backend**: Node.js + TypeScript + Express + tRPC, WebSocket(실시간 동기화), Manus 내장 API 활용.
*   **Database**: PostgreSQL (JSONB 활용 [Policy-DB-001]), Drizzle ORM으로 타입 안전성 확보.
*   **Storage**: Manus 내장 S3 스토리지 (이미지 및 원본 PDF 저장).
*   **호환성**: Xodo 등 외부 툴에서 읽기 가능한 표준 Annotation 객체(FreeText, Stamp, Link 등) 사용.
